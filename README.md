## Getting Started
### Prerequisites
To run the scripts and reproduce the analysis, you will need:
*   **Python 3.x:** (e.g., Python 3.8+)
*   **Jupyter Notebook:** For running the `.ipynb` files.
*   **Git:** For cloning this repository.
*   **Required Python packages:** Listed in `requirements.txt`.
*   **External Bioinformatics Tools:**
    *   **PCCD (Protein Charged Cluster Detector):** Our custom tool used for initial CC screening (As this is a custom tool, further details may be provided).
    *   **IUPred2A:** Used for intrinsic disorder and binding-induced order prediction (using ANCHOR2): Accessed via web server.
    *   **Muscle v5:** For multiple sequence alignment.
    *   **IQ-TREE:** For phylogenetic tree inference.

### Installation and Setup

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/SyrineNebli/Mitochondrial_Charged_Clusters_Plants.git
    cd Mitochondrial_Charged_Clusters_Plants
    ```

2.  **Create a Python virtual environment (recommended):**
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Linux/macOS
    # venv/Scripts/activate   # On Windows
    ```

### Running the Analysis

The core analytical steps are outlined in the Jupyter Notebooks located in the `src/` directory.

1.  **Start Jupyter Notebook:**
    ```bash
    jupyter notebook
    ```
2.  Navigate to the `src/` directory and open `sm2_script_name.ipynb` and `sm4_script_name.ipynb`.
3.  Follow the instructions within each notebook to execute the code cells. These notebooks will guide you through:
    *   Identification of Conserved Charged Clusters (CCC)
    *   Disorder and Binding Prediction
    *   Functional Annotation
    *   Preparation for Phylogenetic Analysis

**Note:** The results presented in the `results/` folder correspond to the outputs generated by running these scripts and external tools as described in the manuscript.

## Data

The `data/` directory contains `supplementary_material_1.json`, which is a JSON export of our custom MongoDB database. This file details each detected Charged Cluster, including:

*   **NCBI Protein database accession number:** Unique identifier for the protein sequence.
*   **Protein sequence’s length:** Total length of the original protein.
*   **CC’s start and end position:** Coordinates of the Charged Cluster within the protein.
*   **CC’s sequence:** The amino acid sequence of the Charged Cluster.
*   **Window size (w):** The window size (15, 20, 25, or 30) used by PCCD for detecting that specific CC.
*   **Statistical significance (p-value):** The p-value associated with the CC's detection.

## Results Outputs

The `results/` directory stores the raw textual outputs from key bioinformatics tools used in our analysis, serving as the source for the figures and tables presented in the manuscript.

*   ***IQ-TREE outputs (`results/IQTree/`): This folder contains the IQ-TREE report files (.iqtree) generated from the multiple sequence alignment using IQ-TREE2 (iqtree2.exe -s NCCC_inter_proteins.afa -B 1000 -alrt 1000). These reports include detailed model information, bootstrap support, and approximate likelihood-ratio test (aLRT) values, and were used to infer the phylogenetic relationships of proteins containing conserved charged clusters (CCC).
*   **IUPRED2A outputs (`results/IUPRED2A/`):** Includes the raw prediction results for intrinsic disorder for various categories of NCCC and PCCC proteins. These files underpin our disorder ratio analyses.
